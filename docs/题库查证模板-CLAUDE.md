# 题库查证规范（通用模板）

## 查证方法

每题从三个维度独立验证，缺一不可：

1. **题目与选项的匹配性** — 选项内容是否与题目相关、选项之间是否构成合理干扰项、选项名称/人物是否真实存在
2. **正确选项与解析的一致性** — 标记的正确答案是否与解析文字描述一致、解析是否在解释正确答案
3. **正确选项的事实正确性** — 答案在事实上是否正确、解析中的引文/出处/年代是否准确

## 常见错误模式

| 类型 | 表现 | 检测方法 |
|------|------|----------|
| 答案解析矛盾 | 答案标记 A，解析描述的是 C | 对比 answer 字段和 explanation 内容 |
| 选项张冠李戴 | 选项与题目主题无关 | 检查每个选项是否在题目语境下合理 |
| 假选项充数 | 使用不存在的书名/人名 | 搜索引擎验证可疑选项 |
| 解析答非所问 | 解析讲的是另一道题 | 确认 explanation 中的人名/作品与题目一致 |
| 年代/朝代错误 | 作品归入错误时期 | 逐条核实年代信息 |
| 引文归属错误 | A 作者的诗归于 B | 检索关键诗句确认归属 |

## 操作流程

1. 批量读取题目（每次 20-50 题）
2. 逐题过三关验证
3. 发现问题立即记录（ID + 问题描述 + 修正方案）
4. 批量修正后复核相邻题目是否受影响
5. 修正 answer 后必须同步修正 explanation

## 数据格式

```json
{
  "id": 1,
  "question": "题目文本",
  "options": ["A. 选项一", "B. 选项二", "C. 选项三", "D. 选项四"],
  "answer": "A",
  "explanation": "解析文本"
}
```
